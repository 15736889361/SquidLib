001package squidpony.squidmath;
002
003import squidpony.StringKit;
004
005import java.io.Serializable;
006
007/**
008 * A quasi-random number generator that goes through one of many sub-random sequences found by J.G. van der Corput.
009 * More specifically, this is a kind of scrambled van der Corput sequence, where the state is internally stored in a
010 * simple 64-bit long that is incremented once per generated number, but the state is altered with a Gray code before
011 * being used, more on this later if you want to read about it. The important things to know about this class are:
012 * size of state affects speed (prefer smaller seeds, but quality is sometimes a bit poor at first if you start at 0);
013 * the base (when given) should be prime and moderately small; this doesn't generate very random numbers (which can be
014 * good for making points that should not overlap); and this is a StatefulRandomness with an additional method for
015 * generating quasi-random doubles, {@link #nextDouble()}.
016 * <br>
017 * This generator allows a base (also called a radix) that changes the sequence significantly; a base should be prime,
018 * and this performs a little better in terms of time used with larger primes, though quality is also improved by
019 * preferring primes that aren't very large relative to the quantity of numbers expected to be generated. Unfortunately,
020 * performance is not especially similar to conventional PRNGs; smaller (positive) state values are processed more
021 * quickly than larger ones, or most negative states. At least one 64-bit integer modulus and one 64-bit integer
022 * division are required for any number to be produced, and the amount of both of these relatively-non-cheap operations
023 * increases linearly with the number of digits (in the specified base, which is usually not 10) needed to represent
024 * the current state. Since performance is not optimal, and the results are rather strange anyway relative to PRNGs,
025 * this should not be used as a direct substitute for a typical RandomnessSource (however, it is similar to, but simpler
026 * than, {@link squidpony.squidmath.SobolQRNG}). So what's it good for?
027 * <br>
028 * A VanDerCorputSequence can be a nice building block for more complicated quasi-randomness, especially for points in
029 * 2D, 3D, or even higher dimensions. Using a VanDerCorputQRNG with base 11 for the x-axis and another VanDerCorputQRNG
030 * with base 19 for the y-axis, requesting a double from each to make points between (0.0, 0.0) and (1.0, 1.0), has an
031 * interesting trait that can be desirable for many kinds of positioning in 2D: once a point has been generated, an
032 * identical point will never be generated until floating-point precision runs out, but more than that, nearby points
033 * will almost never be generated for many generations, and most points will stay at a comfortable distance from each
034 * other if the bases are different and both prime (as well as, more technically, if they share no common denominators).
035 * This is also known as a Halton sequence, which is a group of sequences of points instead of simply numbers. The
036 * choices of 11 and 19 are just examples; any two different primes will work well for 2D, especially prime numbers
037 * between 10 and 60 or so. Three VanDerCorputQRNG sequences could be used for a Halton sequence of 3D points, using
038 * three different prime bases, four for 4D, etc. SobolQRNG can be used for the same purpose, but the points it
039 * generates are typically more closely-aligned to a specific pattern, the pattern is symmetrical between all four
040 * quadrants of the square between (0.0, 0.0) and (1.0, 1.0) in 2D, and it probably extends into higher dimensions.
041 * Using one of the many possible Halton sequences gives some more flexibility in the kinds of procedural random-like
042 * points produced.
043 * <br>
044 * Gray codes sound much more complex than they actually are; the sequence of Gray codes for the integers from 0 to 16
045 * is {@code 0, 1, 3, 2, 6, 7, 5, 4, 12, 13, 15, 14, 10, 11, 9, 8, 24}, and this can be generated efficiently with
046 * {@code i ^ (i >> 1)} *see note. Gray codes, as we use them here, simply are sequences of numbers where exactly one
047 * bit changes between the binary representations of successive Gray codes. There is always a unique Gray code for any
048 * non-negative integer, and all non-negative integers are possible as Gray codes. Because of this, changing the state
049 * via Gray code gives a moderately-good scramble without significantly reducing the total amount of numbers a
050 * VanDerCorputQRNG can generate before repeating.
051 * <br>
052 * Expected output for {@link #nextDouble()} called 33 times on an instance made with {@code new VanDerCorputQRNG(11, 83L)}:
053 * 0.4552967693463561, 0.546205860255447, 0.3643876784372652, 0.2734785875281743, 0.628099173553719,
054 * 0.71900826446281, 0.9008264462809917, 0.8099173553719008, 0.4462809917355372, 0.5371900826446281,
055 * 0.35537190082644626, 0.2644628099173554, 0.3305785123966942, 0.4214876033057851, 0.6033057851239669,
056 * 0.512396694214876, 0.8760330578512396, 0.9669421487603306, 0.7851239669421488, 0.6942148760330579,
057 * 0.4297520661157025, 0.5206611570247934, 0.7024793388429752, 0.6115702479338843, 0.24793388429752067,
058 * 0.33884297520661155, 0.15702479338842976, 0.06611570247933884, 0.5950413223140496, 0.6859504132231405,
059 * 0.8677685950413223, 0.7768595041322314, 0.1487603305785124
060 * <br>
061 * Expected output for {@link #nextDouble()} called 33 times on an instance made with {@code new VanDerCorputQRNG(19, 83L)}:
062 * 0.6481994459833795, 0.7008310249307479, 0.5955678670360111, 0.5429362880886427, 0.12188365650969529,
063 * 0.1745152354570637, 0.27977839335180055, 0.22714681440443213, 0.01662049861495845, 0.06925207756232687,
064 * 0.961218836565097, 0.9085872576177285, 0.22160664819944598, 0.2742382271468144, 0.37950138504155123,
065 * 0.3268698060941828, 0.5373961218836565, 0.590027700831025, 0.48476454293628807, 0.43213296398891965,
066 * 0.853185595567867, 0.9058171745152355, 0.013850415512465374, 0.9584487534626038, 0.7479224376731302,
067 * 0.8005540166204986, 0.6952908587257618, 0.6426592797783933, 0.7977839335180056, 0.850415512465374,
068 * 0.9556786703601108, 0.9030470914127424, 0.11634349030470914
069 * <br>
070 * Note on Gray code implementation: typically the unsigned right shift would be used instead of the example code, which
071 * could look like {@code i ^ (i >>> 1)}, but using the sign-extending right shift here has a useful property of making
072 * this only output positive Gray codes.
073 * Created by Tommy Ettinger on 11/18/2016.
074 */
075public class VanDerCorputQRNG implements StatefulRandomness, RandomnessSource, Serializable {
076    private static final long serialVersionUID = 1;
077    public long state;
078    public final int base;
079
080    /**
081     * Constructs a new van der Corput sequence generator with base 13 and starting point 83.
082     */
083    public VanDerCorputQRNG()
084    {
085        base = 13;
086        state = 83;
087    }
088
089    /**
090     * Constructs a new van der Corput sequence generator with the given starting point in the sequence as a seed.
091     * Usually seed should be at least 20 with this constructor, but not drastically larger; 2000 is probably too much.
092     * This will use a base 13 van der Corput sequence.
093     * @param seed the seed as a long that will be used as the starting point in the sequence; ideally positive but low
094     */
095    public VanDerCorputQRNG(long seed)
096    {
097        base = 13;
098        state = seed;
099    }
100
101    /**
102     * Constructs a new van der Corput sequence generator with the given base (a.k.a. radix; if given a base less than
103     * 2, this will use base 2 instead) and starting point in the sequence as a seed. Good choices for base are between
104     * 10 and 60 or so, and should usually be prime. Good choices for seed are larger than base but not by very much,
105     * and should generally be positive at construction time.
106     * @param base the base or radix used for this VanDerCorputQRNG; for most uses this should be prime but small-ish
107     * @param seed the seed as a long that will be used as the starting point in the sequence; ideally positive but low
108     */
109    public VanDerCorputQRNG(int base, long seed)
110    {
111        this.base = base < 2 ? 2 : base;
112        state = seed;
113    }
114
115    /**
116     * Gets the next quasi-random long as a fraction of {@link Long#MAX_VALUE}; this can never produce a negative value.
117     * It is extremely unlikely to produce two identical values unless the state is very high or is negative; state
118     * increases by exactly 1 each time this, {@link #next(int)}, or {@link #nextDouble()} is called and can potentially
119     * wrap around to negative values after many generations.
120     * @return a quasi-random non-negative long; may return 0 rarely, probably can't return {@link Long#MAX_VALUE}
121     */
122    @Override
123    public long nextLong() {
124        long s = ++state ^ (state >> 1), // intentionally non-standard Gray code; ensures s is non-negative
125                num = s % base, den = base;
126        while (den <= s) {
127            num *= base;
128            num += (s % (den * base)) / den;
129            den *= base;
130        }
131        return (Long.MAX_VALUE / den) * num;
132    }
133
134    @Override
135    public int next(int bits) {
136        return (int)(nextLong()) >>> (32 - bits);
137    }
138
139    /**
140     * Gets the next quasi-random double from between 0.0 and 1.0 (normally both exclusive; only if state is negative or
141     * has wrapped around to a negative value can 0.0 ever be produced). It should be nearly impossible for this to
142     * return the same number twice unless floating-point precision has been exhausted or a very large amount of numbers
143     * have already been generated. Certain unusual bases may make this more likely.
144     * @return a quasi-random double that will always be less than 1.0 and will be no lower than 0.0
145     */
146    public double nextDouble() {
147        long s = ++state ^ (state >> 1), // intentionally non-standard Gray code; ensures s is non-negative
148                num = s % base, den = base;
149        while (den <= s) {
150            num *= base;
151            num += (s % (den * base)) / den;
152            den *= base;
153        }
154        return num / ((double)den);
155    }
156
157    @Override
158    public VanDerCorputQRNG copy() {
159        return new VanDerCorputQRNG(base, state);
160    }
161
162    @Override
163    public long getState() {
164        return state;
165    }
166
167    @Override
168    public void setState(long state) {
169        this.state = state;
170    }
171
172    @Override
173    public String toString() {
174        return "VanDerCorputQRNG with base " + base +
175                " and state 0x" + StringKit.hex(state);
176    }
177
178    @Override
179    public boolean equals(Object o) {
180        if (this == o) return true;
181        if (o == null || getClass() != o.getClass()) return false;
182        VanDerCorputQRNG that = (VanDerCorputQRNG) o;
183        return state == that.state && base == that.base;
184    }
185
186    @Override
187    public int hashCode() {
188        int result = (int) (state ^ (state >>> 32));
189        result = 31 * result + base;
190        return result;
191    }
192}
